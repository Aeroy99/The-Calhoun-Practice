# Sanity Free Option (Reference)

This project is currently configured with Decap CMS.

If you later choose Sanity instead, this is the free-tier path to evaluate.

## Why consider Sanity

- Managed editor experience
- Strong content modeling tools
- Good editor UX for non-technical users

## Free-tier note

Sanity publicly advertises a Free plan (`$0`) with plan limits.
Always re-check current pricing/limits directly before final commitment.

## Migration impact from Decap

Switching to Sanity later would require:

1. Creating Sanity project + dataset.
2. Modeling the same homepage fields.
3. Replacing `public/content/home-content.json` fetch with Sanity client fetch.
4. Migrating existing content values.

## Recommendation

Use Decap now for zero-cost launch and review partner editing comfort.
If editor UX becomes the main pain point, migrate to Sanity in a dedicated PR.
